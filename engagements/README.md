# Engagements

Each run of `/solve-case` creates a folder here named after a short
kebab-case slug for the case (e.g. `regional-grocery-margin/`). Nothing is
shared between engagements — each is self-contained.

## Files in each engagement folder

| File | Produced by | Contents |
|---|---|---|
| `01-intake.md` | `case-classifier` | Case archetype, the real question, known facts, critical unknowns |
| `02-framework.md` | `framework-strategist` | Issue tree, framework rationale, specialist dispatch plan |
| `03-<specialist>.md` | `financial-analyst` / `market-analyst` / `operations-analyst` | One file per specialist actually dispatched |
| `04-challenge.md` | `challenger` | Stress-test memo and verdict |
| `report.md` | `report-writer` | The final executive deliverable — this is the one to read |

If a specialist gets re-dispatched after a "needs rework" verdict from the
challenger, its file is overwritten with the revised analysis — the
challenge memo and report reflect the final version only.
