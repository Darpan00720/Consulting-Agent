# Sample Telemetry Traces

Three committed traces, one per pilot engagement, generated by
`scripts/replay_pilots.py`. The span data is **reconstructed from the real pilot
run logs** — durations, token counts, and governance verdicts actually observed
when Northwind, Halberd, and Harbor & Vine were executed (see
[v1.0-Validation-Campaign](../../reviews/v1.0-Validation-Campaign.md)).

| File | Engagement | Spans | Notable |
|---|---|---|---|
| `eng_northwind_eu.jsonl` | Market entry (full pipeline) | 15 | 5 analysts, reviewer+challenger, gate |
| `eng_halberd_cost.jsonl` | Cost reduction (lightweight + rework) | 8 | 1 reworked span; challenger needs_rework→stands |
| `eng_harbor_vine_org.jsonl` | Org redesign (lightweight) | 6 | 4 frameworks |

Regenerate: `uv run python scripts/replay_pilots.py`

> **Replay caveat (honest):** these are replays, so every event is emitted at
> once — `total_wall_ms` is ~milliseconds (the replay time), **not** the original
> engagement wall-clock. The real observed durations live in `active_ms` and
> `duration_by_phase_ms` (sums of the recorded span durations). On a **live**
> run, real timestamps make `total_wall_ms` meaningful.
