# M1.7.4 Design — Replay Integrity

## 1. Objective

Define — and in the implementation phase, build — the **replay integrity
checker**: the single verification gate that an event log (and, when present,
a paired state snapshot) must pass before it may be folded by `project()` or
used to construct an append-capable engagement.

Why this exists, from the approved contract [Verified]:
- M1.7-Design D2: "any component that feeds a log to `project()` … must first
  pass it through an **integrity check** — `events[i].metadata.seq == i + 1`
  for all i — and refuse to fold on violation (`ReplayIntegrityError`)."
- M1.7-Design D4: the ADR-002 invariant `state_version == max(seq)` "is then
  enforced at the **boundary** … the replay/rebuild integrity check of D2
  verifies it alongside contiguity."
- ADR-002 (§Why event sourcing): "deterministic replay", "old events always
  remain replayable"; §26: the log is the append-only system of record;
  the R/W matrix marks events "none (immutable)".
- The append pipeline guarantees these properties **at write time** (P1–P24)
  [Verified: 206-test suite]. Nothing yet guarantees them **at rest** — a log
  arriving from persistence (M1.8), a snapshot restore, or any external source
  is unverified. M1.7.4 closes exactly that gap.

Consumers [Verified from M1.7-Design D2 + M1.7.3-S5]: M1.8 load, M1.9 replay
(which will construct `AppendPipeline(state, log=…, append_supported=True)` —
the parameter and capability flag already exist for this), recovery, and
snapshot restore. M1.7.4 ships the checker and its error taxonomy; the
consumers arrive in M1.8/M1.9.

## 2. Scope

- The **replay integrity invariants** (§4) as an executable, pure, IO-free
  verification function over `(log)` and `(log, snapshot-expectations)`.
- The **`ReplayIntegrityError` taxonomy** (§5) with stable machine-readable
  codes (the S1 `error_code` discipline — a frozen, additive-only namespace).
- Placement (design decision for approval): `packages/state/append/integrity.py`
  — the log machinery lives with the log; M1.9's replay package consumes it.
  Internal module; **no public API additions** (surfaced by M1.8/M1.9 when the
  consumers exist), consistent with this milestone's "no new APIs" rule.

## 3. Out of Scope

- Persistence, file formats, byte-level corruption, fsync ordering — M1.8
  [Verified: M1-Decomposition assigns "append / save / load" to M1.8].
- Replay execution (rebuild, projection-driven construction, snapshots,
  recovery) — M1.9 [Verified: M1-Decomposition §M1.9].
- Event schema migration / upcasting — M1.8 (TD-007) [Verified: checkpoint
  TD register; no persisted events exist pre-M1.8, so no old event can exist].
- Any change to projection (stays seq-blind [Verified: projection.py folds in
  iteration order, never reads seq for ordering]), validation, the guard, the
  pipeline, the facade, or any ADR.

## 4. Replay Integrity Invariants

Log-only invariants (verifiable from the log alone):

| ID | Invariant | Rationale | Severity | Evidence | Class |
|---|---|---|---|---|---|
| **R1** | **Sequence origin:** a non-empty log's first event has `seq == 1` | D2: "seq starts at 1"; the allocator's first stamp is `current_sequence(()) == 1` | fatal | M1.7-Design D2; S2 tests C1/C2; Events.md sequence contract | [Verified] |
| **R2** | **Contiguity:** `log[i].seq == i + 1` for all i — no gaps | D2's literal check; gaps mean lost events (a failed append never consumes a seq, so a gap cannot be innocent) | fatal | M1.7-Design D2; P7 (failures consume no seqs); S2 A1 | [Verified] |
| **R3** | **Sequence uniqueness** — no duplicate seq | Duplicate seq breaks total order and optimistic concurrency (ADR-002 §26: seq = "total order + optimistic concurrency") | fatal | ADR-002 §26; implied by R2 but detected distinctly for diagnosis | [Verified] |
| **R4** | **Ordering authority:** list order equals seq order (strictly ascending) | D2: "the committed log's seq is the single source of truth for ordering" — a disordered list contradicts its own seqs | fatal | M1.7-Design D2; Events.md sequence contract | [Verified] |
| **R5** | **No unassigned events:** no `seq == 0` in a committed log | `0` is the unassigned sentinel; a committed log can only contain stamped copies | fatal | D2 sentinel rule; S2 A7 (only unassigned stampable); guard G10 | [Verified] |
| **R6** | **Event-id uniqueness** across the log | Idempotency: an `event_id` is committed at most once per engagement | fatal | Guard G11/G12; Events.md idempotency clause | [Verified] |
| **R7** | **Engagement identity uniformity:** every event's `engagement_id` is identical | One log = one engagement's history; the guard rejects foreign/mixed batches at write time (G9/G16), so a mixed log at rest is corruption | fatal | Guard G9/G16; ADR-002 §26 (`engagement_id` = scope) | [Verified] |
| **R8** | **Genesis:** a non-empty log's first event is `EngagementCreated` | ADR-002's lifecycle makes `EngagementCreated` every engagement's first event; the reducer rebuilds metadata from it — without it, replay folds onto the placeholder identity (empty `engagement_id`, epoch timestamps) and yields an identity-less state that, notably, can accept no appends (the guard would reject identity-bearing events as foreign) | fatal (for replay-to-engagement) | ADR-002 §State Lifecycle (first ES append is EngagementCreated); projection `_initial_state()` placeholder [Verified in code]; P11 convergence note; S5 finding that placeholder pipelines are inert | [Inference] — ADR-002 shows it in the lifecycle diagram but never states "MUST be first" as a rule; adopted here as a checked invariant, flagged for approval |
| **R9** | **Deterministic replay:** the same log always folds to the same state | The entire basis of event sourcing here (ADR-002: "deterministic replay"); M1.5 proved it and M1.7.2 extended it to `state_version` | precondition (not checked — guaranteed by projection) | M1.5 determinism tests; `test_reprojection_yields_identical_state_version`; P14 | [Verified] |
| **R10** | **Unknown-but-parseable events are not integrity failures** | Projection's default reducer is a deliberate no-op ("projection never fails") while `apply` still stamps `state_version` — forward-compatible logs are an ADR-002 §Design-Principle ("old events always remain replayable"), not corruption | n/a (documented behavior) | projection.py `_apply` default [Verified in code]; ADR-002 line 76 | [Verified] |

Paired invariants (verifiable only against a snapshot / expectation — e.g. a
persisted `(state, log)` pair at M1.8 load):

| ID | Invariant | Rationale | Severity | Evidence | Class |
|---|---|---|---|---|---|

| **R15** | **Replay across projection versions:** replaying any valid log under the current implementation yields a `projection_version == PROJECTION_VERSION` state; the old snapshot is discarded, not merged | Policy doc migration rule: "re-project any `projection_version = 1` state from its log" | n/a (consequence, not check) | projection-versioning.md §Version history | [Verified] |
| **R16** | **Schema-version handling:** every event today carries `schema_version == 1` [Verified: EventMetadata default; no writer sets anything else]. | **R17** | **Replay after migration:** no migration mechanism exists in the repository | Cannot verify what does not exist; M1.8 defines storage migration, and its design must add the corresponding integrity codes then (the code namespace is additive-frozen for exactly this) | deferred | absence verified by repository search | [Unknown] |

Empty-log note [Verified]: an empty log is **valid** — `project([])` returns
the empty state (M1.5 approved contract) and `current_version(()) == 0`; R1/R8
apply only to non-empty logs.

Byte-level corruption note [Verified boundary]: the checker operates on parsed
`Event` objects. Undeserializable bytes never reach it — the discriminated
`Event` union rejects unknown `type`s and malformed payloads at parse time
(M1.4), and parse failures are M1.8 persistence errors, not replay-integrity
errors.

## 5. Failure Taxonomy — `ReplayIntegrityError` hierarchy

Follows the S1 error discipline exactly [Verified precedent]: a frozen,
additive-only `StrEnum` of stable codes; human-readable messages that are
never a contract; errors extend `StratAgentError` (per M1.7-Design §3, which
places `ReplayIntegrityError` outside the `AppendError` family — these are not
append failures).

```
ReplayErrorCode (StrEnum, frozen namespace)      severity      trigger
──────────────────────────────────────────────────────────────────────────────
SEQUENCE_ORIGIN        "sequence_origin"         fatal         first seq != 1
SEQUENCE_GAP           "sequence_gap"            fatal         seq[i+1] > seq[i] + 1
SEQUENCE_DUPLICATE     "sequence_duplicate"      fatal         seq[i+1] == seq[i]
SEQUENCE_DISORDER      "sequence_disorder"       fatal         seq[i+1] < seq[i]
UNASSIGNED_EVENT       "unassigned_event"        fatal         seq == 0 in the log
EVENT_ID_DUPLICATE     "event_id_duplicate"      fatal         event_id seen twice
ENGAGEMENT_MISMATCH    "engagement_mismatch"     fatal         mixed engagement_ids, or paired snapshot's id differs
GENESIS_MISSING        "genesis_missing"         fatal         non-empty log's first event is not EngagementCreated
STATE_VERSION_MISMATCH "state_version_mismatch"  fatal         paired snapshot.state_version != last seq (also how truncation surfaces, R13)
PROJECTION_FUTURE      "projection_future"       fatal         snapshot.projection_version > PROJECTION_VERSION
PROJECTION_STALE       "projection_stale"        recoverable   snapshot.projection_version < PROJECTION_VERSION
```

Hierarchy:

- `ReplayIntegrityError(StratAgentError)` — base; carries `error_code:
  ReplayErrorCode`, the offending position (`index: int | None`), offending
  `event_id: str | None`, and structured payload fields per code (e.g.
  `expected`/`actual` seq values). One concrete class per *family* rather than
  eleven classes: `SequenceIntegrityError` (the five sequence codes),
  `LogIdentityError` (event-id / engagement / genesis codes),
  `SnapshotMismatchError` (the three paired codes) — keeping `isinstance`
  useful without a class explosion; the **code** is the precise contract.
- **Recovery guidance** (documented on each class and in the API docs):
  - all `SEQUENCE_*`, `EVENT_ID_DUPLICATE`, `ENGAGEMENT_MISMATCH`,
    `GENESIS_MISSING`, `STATE_VERSION_MISMATCH`: the artifact is corrupt or
    incomplete — do not fold; recover from the persistence layer (M1.8:
    earlier snapshot + longer log, backups) or human intervention. Never
    "repair" by renumbering, dropping, or synthesizing events (the S5
    no-fabrication rule applies at rest exactly as it does at append).
  - `PROJECTION_STALE`: deterministic remediation — discard the snapshot,
    re-project the (verified) log under the current implementation.
  - `PROJECTION_FUTURE`: refuse; requires a newer code version to read.

## 6. Validation Algorithm (pseudo-code only)

```
verify_log(log) -> None | raise ReplayIntegrityError
    if log is empty: return                        # valid: empty state, version 0
    if log[0] is not EngagementCreated:            raise GENESIS_MISSING(index=0)
    if log[0].seq != 1:
        if log[0].seq == 0:                        raise UNASSIGNED_EVENT(index=0)
        else:                                      raise SEQUENCE_ORIGIN(index=0)
    engagement = log[0].engagement_id
    seen_ids   = { log[0].event_id }
    for i in 1 .. len(log)-1:
        e = log[i]
        if e.engagement_id != engagement:          raise ENGAGEMENT_MISMATCH(index=i)
        if e.seq == 0:                             raise UNASSIGNED_EVENT(index=i)
        if e.seq == log[i-1].seq:                  raise SEQUENCE_DUPLICATE(index=i)
        if e.seq <  log[i-1].seq:                  raise SEQUENCE_DISORDER(index=i)
        if e.seq >  log[i-1].seq + 1:              raise SEQUENCE_GAP(index=i)
        if e.event_id in seen_ids:                 raise EVENT_ID_DUPLICATE(index=i)
        seen_ids.add(e.event_id)

verify_pair(log, snapshot) -> None | raise
    verify_log(log)
    if snapshot.projection_version >  PROJECTION_VERSION:   raise PROJECTION_FUTURE
    if log non-empty and snapshot.engagement_id != log[0].engagement_id:
                                                   raise ENGAGEMENT_MISMATCH
    expected = last seq (0 if empty)               # via S2 current_version(log)
    if snapshot.state_version != expected:         raise STATE_VERSION_MISMATCH
    if snapshot.projection_version <  PROJECTION_VERSION:   raise PROJECTION_STALE
```

Properties: pure, deterministic, IO-free, single pass, first-failure with a
precise index (mirroring the S3 guard's deterministic-order discipline); reads
only event metadata plus the snapshot's three identity/version fields; the one
arithmetic-adjacent fact (`expected version`) is delegated to S2
`current_version` (ownership discipline, P17 spirit). Note the deliberate
check order: within the loop, identity → sentinel → order → gap → id, so a
given corrupt log always raises the same code.

## 7. Complexity Analysis

| | Time | Memory |
|---|---|---|
| `verify_log` | **O(n)** single pass — worst = expected (no early structure to exploit; a valid log must be fully scanned) | **O(n)** for the event-id set; all else O(1) |
| `verify_pair` | O(n) (dominated by `verify_log`; the pair checks are O(1)) | O(n) |
| Failure case | O(k) to first defect at index k (first-failure) | O(k) |

At engagement scale (≤ ~10⁴ events, per the projection baselines) this is
sub-millisecond and runs once per load/replay, not per append [Inference from
the measured projection baselines: a full fold of 10⁴ events costs ~33 ms;
a metadata-only scan is strictly cheaper].

## 10. Risks

**Blocking** — none identified. The checker touches no frozen code and has no
consumers to break.

**Medium**
- **Dual definitions drift** (guard vs checker): both enforce "well-formed
  log" facts; if a future change updates one and not the other, write-time and
  at-rest semantics diverge. Mitigation: R10's cross-check test (a
  pipeline-produced log must pass the checker) makes drift loud; this document
  is the single invariant definition both cite.
- **R8 (genesis) is [Inference], not ADR-verbatim.** If the user rejects it as
  an integrity invariant, identity-less replays become representable and M1.9
  must handle placeholder-identity engagements. Flagged for explicit approval.
- **M1.8 unknowns**: the persisted-pair shape is undefined; `verify_pair` is
  designed against the only stable contracts that exist (`EngagementState`
  metadata fields + the log). If M1.8's format diverges, the checker's *codes*
  stay stable and only its call site adapts.

**Low**
- Performance (O(n) scan, run once per load) — negligible at scale.
- Taxonomy granularity (5 sequence codes vs 1): chosen for recovery precision;
  cost is a slightly wider frozen namespace.
