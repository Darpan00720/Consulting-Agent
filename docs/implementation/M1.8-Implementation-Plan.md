# M1.8 Implementation Plan — Persistence

Approved decisions incorporated: **DD-1** sibling package `packages/persistence/`
(state stays IO-free); **DD-2** minimal manifest (persistence metadata only);
**DD-3** new `EngagementStore` (facade unchanged); **DD-4** persistence owns the
`engagements/<slug>/` layout exclusively; **DD-5** reject torn writes (no repair);
**DD-6** M1.8 = save/load/integrity/validation, M1.9 = repair/migration/replay;
**DD-7** explicit persistence only; **PER-011** deterministic persistence.

## 6. Why Each File Changes

- **errors.py** — S4 raises typed persistence failures; defining them first lets
  every slice code against fixed contracts.
- **paths.py** — DD-4: one place owns the FS layout; nothing else may know it.
- **format.py** — isolates the on-disk format from IO so PER-011 determinism and
  round-trip are testable without touching the filesystem.
- **atomic.py** — isolates fsync/rename so atomicity (PER-005/006) is auditable
  in one module.
- **store.py** — the composition point; the only module importing both
  `state.append` seams and the IO/codec modules.
- **.gitignore** — DD-4: persisted engagement data is runtime state, not source.

## 7. Public vs Internal APIs

- **Public (persistence package `__all__`):** `EngagementStore`;

- **Internal to the package:** `paths`, `format`, `atomic` (implementation).
- **Consumed from `state` (unchanged):** `Engagement`, `AppendPipeline`,

## 8. Storage Layout (owned solely by `persistence.paths` — DD-4)

```
engagements/<slug>/
├── events.log      append-only NDJSON — one Event per line, seq order
├── state.json      EngagementState snapshot (facade to_json output)
└── manifest.json   persistence metadata only (format + integrity)
```
`<slug>` = `EngagementState.metadata.slug` [Verified field]. Validated against a
strict pattern (kebab-case, no path separators) to prevent traversal.

## 9. Manifest Schema (DD-2 — persistence metadata only, no duplication)

```json
{ "format_version": 1,
  "log_sha256": "<hex of events.log bytes>",
  "snapshot_sha256": "<hex of state.json bytes>" }
```
- **No duplication** [DD-2]: engagement_id/slug/state_version/projection_version
  live in `state.json`; event count / last seq live in `events.log`. The manifest
  carries only what is *not* elsewhere: the store's own `format_version` and the
  two content checksums (integrity).
- **Fully deterministic** → PER-011 needs **no** "non-deterministic field"
  exception: the manifest is a pure function of the two files' bytes; no
  wall-clock timestamp is stored. [Resolves the Phase-1 `written_at` concern.]
- **Purpose:** the manifest, written last, is the commit marker — a present,
  checksum-matching manifest means log+snapshot are already durable and
  consistent (torn-write detection, DD-5).

## 10. NDJSON Event Schema (`events.log`)

One line per event = `event.model_dump_json()` (the discriminated `Event` union;
`type` field selects the concrete model). Read per line via
`TypeAdapter(Event).validate_json(line)` [Verified round-trips, types+seq
preserved]. Lines are in `seq` order (= append order). `\n`-terminated, UTF-8.

## 11. Snapshot Schema (`state.json`)

`EngagementState.model_dump_json()` — byte-identical to `Engagement.to_json()`
[Verified] and documented by the generated `schema/engagement-state.schema.json`
(drift-tested). Deterministic field order [Verified — same-instance byte-equal].

## 12. Atomic Save Workflow

`save(engagement)` — full, idempotent durable write. Let `committed` = the
engagement's committed `(log, state)` (read via the pipeline, P-DD-A):

1. **Serialize** (pure, `format.py`): `log_bytes` (NDJSON), `snapshot_bytes`.
2. **Log** (`atomic.py`): if `events.log` absent → write it; if a proper prefix
   exists → append only the missing lines; else (divergent prefix) → raise
   `TornWriteError` (no rewrite of committed history). `flush + fsync(file)`.
3. **Snapshot**: `atomic_write(state.json, snapshot_bytes)` = write `.tmp`,
   `fsync`, `os.replace`, `fsync(dir)`.
4. **Manifest**: compute checksums of the now-durable files; `atomic_write(
   manifest.json, …)`.

**Ordering (PER-006):** log → snapshot → manifest, each fully durable before the
next. The manifest is the last write, so its presence+match ⇒ a consistent set.
Complexity **O(n)** (n = total events; the prefix check reads the existing log).

## 13. Atomic Load Workflow

`load(slug) -> Engagement`:

1. Resolve paths (`paths.py`); missing dir/files → `MissingArtifactError`.
2. Read bytes; **verify manifest checksums** vs the files → mismatch =
   `TornWriteError`/`CorruptArtifactError` (DD-5, no repair).
3. Parse: `events.log` → `tuple[Event]` (per-line; a non-parseable/torn trailing
   line → `CorruptArtifactError`, reject — P-DD not needed, DD-5); `state.json`
   → `EngagementState`.
4. **`verify_log(log)`** then **`verify_pair(log, state)`** [Verified seams] →
   any `ReplayIntegrityError` propagates (e.g. `STATE_VERSION_MISMATCH`,
   `PROJECTION_FUTURE/STALE`). M1.8 refuses; M1.9 repairs.
5. Reconstruct: `Engagement(AppendPipeline(state, log=log, append_supported=True))`
   [Verified seam — `__init__` rebuilds `Committed` via `make_committed`].
   → an append-capable engagement whose next append continues the seq (PER-007).

Complexity **O(n + |state|)**: O(n) read + O(n) verify_log + O(|state|) snapshot
parse; **no re-projection** (snapshot trusted *after* verify_pair — that is the
M1.8/M1.9 line, DD-6).

## 14. Failure Matrix

| Failure | Detection | Result (DD-5: reject, no repair) |
|---|---|---|
| Missing dir / file | paths | `MissingArtifactError` |
| Manifest checksum mismatch | load step 2 | `TornWriteError` |
| Torn/partial trailing log line | per-line parse | `CorruptArtifactError` |
| Garbled JSON (log or snapshot) | parse | `CorruptArtifactError` (path+line) |
| `seq` gap/dup/disorder/origin | `verify_log` R1–R5 | `ReplayIntegrityError` |
| Missing/duplicate/late genesis | `verify_log` R8 | `ReplayIntegrityError` |
| Mixed/empty engagement id | `verify_log` R7/R18 | `ReplayIntegrityError` |
| snapshot vs log version mismatch | `verify_pair` R11 | `ReplayIntegrityError` (→ M1.9 rebuild) |
| `projection_version` future | `verify_pair` R14 | `ReplayIntegrityError` (fatal) |
| `projection_version` stale | `verify_pair` R14 | `ReplayIntegrityError` (→ M1.9 re-project) |
| unknown event `schema_version` | codec | `IncompatibleVersionError` (TD-007 defers upcasting) |
| divergent log prefix on save | save step 2 | `TornWriteError` (never rewrite history) |

|---|---|---|

## 16. Stable Invariants (PER-001 … PER-011)

| ID | Invariant | Enforced by |
|---|---|---|
| PER-001 | Round-trip fidelity: `load(save(E))` has identical committed log/state/version | store |
| PER-002 | `events.log` is append-only NDJSON in contiguous seq order | atomic append + verify_log |
| PER-003 | A completed `save` yields a `verify_pair`-passing artifact | save ordering |
| PER-004 | `load` never admits a torn/inconsistent artifact — raises a typed error | verify + checksums |
| PER-005 | Snapshot/manifest writes are atomic (temp→fsync→rename) | atomic.py |
| PER-006 | Durability ordering: log durable before snapshot before manifest | save sequence |
| PER-007 | Loaded engagements are append-capable; seq continues contiguously | AppendPipeline(log=…, append_supported=True) |
| PER-008 | No fabrication/repair on load (no renumber/synthesize/migrate) | store contract |
| PER-009 | Persistence is the only IO; `packages/state` stays IO-free | package boundary + boundary test |
| PER-010 | Unknown `projection_version`/`schema_version` refused | verify_pair + codec |
| PER-011 | **Deterministic persistence:** `save(load(save(E))) == save(E)` byte-identical (manifest fully deterministic; no non-deterministic fields) | codec determinism + full round-trip |

## 19. Risks

- **[Medium] Save incrementality / prefix check** — a full-write `save` is O(n)
  each call; the prefix-append keeps the log append-only but re-reads it. A true
  O(k) incremental append is deferred (noted; not needed for correctness).
- **[Medium] engagements/<slug> shared with the /solve-case plugin** [Verified —
  engagements/README.md writes `01-intake.md`…`report.md`]. M1.8 adds
  `events.log`/`state.json`/`manifest.json` in the same folder; slugs must be
  consistent. Gitignore covers both (runtime artifacts).
- **[Low] fsync/rename portability** — POSIX `os.replace`+dir-fsync assumed;
  Windows differs. Dev target is macOS/Linux [Verified].
- **[Low] P-DD-A coupling** — the store reads `engagement._pipeline.committed()`
  (an internal seam) to preserve the facade freeze; documented, boundary-tested.

## 20. Architectural Concerns

None beyond the risks. Persistence is a sibling package composing frozen seams;
`state` gains no IO; the public surfaces of `state` and the facade are untouched.
The one internal coupling (reading the committed log for `save`) is the minimum
needed and is the subject of remaining decision P-DD-A.
