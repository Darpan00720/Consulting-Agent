## 6. Architecture / Agent / Package / Test / Documentation summaries

- **Architecture:** two-layer — an LLM orchestrator + 16 markdown subagents over a
  deterministic 13-package Python platform (frozen `state`/`persistence`/`replay`
  core; additive `evidence`, `orchestration`, `telemetry`). ADR-driven; RC1.2
  unified the framework store, made governance gates mandatory, and put a
  deterministic validation gate on the live path.
- **Agents (16):** case-classifier, information-gap, planner, framework-selector,
  framework-strategist (legacy), issue-tree-generator, knowledge-agent, financial/
  market/operations/strategy/risk analysts, reviewer, challenger, report-writer,
  knowledge-curator.
- **Packages (13):** common, core, state, persistence, replay, knowledge, planning,
  analysis, governance, reporting, orchestration, evidence, telemetry.
- **Tests (954):** unit + integration + convergence guards + perf baselines +
  telemetry replay; 98.9% coverage.
- **Docs (76):** ADRs, guides (Quickstart/User/Developer), Operations Runbook,
  observability, beta program, reviews, plus the full OSS file set.

## 7. Known Limitations (ship with the release)
1. **Empty evidence base** — every number is a labeled assumption unless supplied;
   evidence-provider seam (ADR-007) is unpopulated. Blocks production; requires
   human review of all numbers.
2. **Non-determinism** — same input can differ across runs (LLM layer).
3. **Small validation sample** — consulting quality proven on 3 genuine engagements.
4. **Flaky perf test** under coverage (R-3); **ADR-001–005 unratified** (Proposed);
   **leaf-detection duplicated ×3** (C-1); **live per-agent telemetry is
   instruction-driven**, not code-enforced. All tracked in
   [ROADMAP](../../ROADMAP.md) and the [Release Audit](../reviews/v1.0-Release-Audit.md).

## 11. Recommended GitHub Release Text

> ### StratAgent v0.1.0-rc2 — First public release candidate
>
> StratAgent is an AI management-consulting platform (a Claude Code plugin on the
> Ruflo harness) that runs a full, **governed** consulting engagement on a business
> problem — classify → scope → frame → analyze → **review → challenge** → validate →
> report — and produces an evidence-traceable, executive-ready deliverable.
>
> **Highlights**
> - Full engagement lifecycle with **16 specialist agents** and one command (`/solve-case`).
> - **Mandatory governance** — Reviewer + Challenger gates + a rework loop; a
>   deterministic validation gate blocks any un-evidenced report.
> - **Evidence discipline** — every number is a client fact or a labeled
>   `[ASSUMPTION]` with a breakeven; labels survive into the report.
> - **Operational telemetry** — per-engagement JSONL traces, analytics, OTel-ready spans.
> - Deterministic Python core; **954 tests**, ruff/black/mypy clean, 98.9% coverage.
>
> **Maturity:** Ready for Limited Beta. StratAgent is decision *support*, not a
> decision-maker — the knowledge base ships no benchmark data, so **a human must
> verify every number**. Not for regulated advice or unsupervised production use.
>
> **Known limitations:** assumption-only evidence (evidence-provider seam ships
> empty), non-deterministic output, and a small (n=3) consulting-quality validation
> sample. See the [Research Evaluation](docs/reviews/v1.0-Research-Evaluation.md)
> and [ROADMAP](ROADMAP.md).
>
> **License:** MIT. **Docs:** start with the [Operations Runbook](docs/operations/Operations-Runbook.md).
