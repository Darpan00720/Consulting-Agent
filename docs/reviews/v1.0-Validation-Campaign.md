# StratAgent v1.0 — Validation Campaign: Final Report

**Independent Consulting QA · Evaluation only (no feature work)**
**Date:** 2026-07-09 · **Build:** `3cb863a` (0.1.0-rc1) · **Model driving agents:** Claude Opus 4.8

---

## 1. Executive Summary

StratAgent was subjected to an independent, end-to-end validation. A 30-case
consulting benchmark and a 13-criterion rubric were built
([v1.0-Validation-Benchmark.md](v1.0-Validation-Benchmark.md)), and the system
was run **genuinely** — real `/solve-case` orchestration dispatching real
specialist subagents — on a representative pilot of **3 cases** spanning a
full pipeline and the sanctioned lightweight pipeline, quantitative and
qualitative cases, and a covered and an uncovered archetype.

**Headline finding: the consulting-reasoning engine and the governance gates
genuinely work.** Across 32 real subagent dispatches, the system produced
board-defensible reasoning, disciplined fact/assumption labeling with **zero
observed hallucinations**, non-obvious senior-level insight (e.g. a CLOUD Act
sovereignty ceiling the analysts had missed), and — most importantly — a
**working governance loop**: the Challenger caught a real cross-analyst
numeric inconsistency, returned `needs_rework`, and the rework loop
(`needs_rework → rework → re-challenge → stands_with_caveats`) resolved it.

**The honest limitations are real but not correctness-breaking.** The
knowledge vault contains *no benchmark data* (by design, D-6), so every number
in every engagement is a labeled assumption — the system reasons well but
cannot ground claims in external evidence. Two divergent framework stores
disagree on coverage. The lightweight pipeline skips the Reviewer, letting a
numeric inconsistency reach the Challenger. And the RC1 Python reporting layer
(`render_report`, `check_render_ready`) is **not on the live engagement
path** — the live report is authored by the LLM `report-writer`, not the
deterministic renderer.

**Aggregate score: 7.5 / 10** (pilot, n=3). **Verdict: READY WITH MINOR
IMPROVEMENTS** — pilot-scoped; a full 30-case run is required for a
production-grade statistical verdict.

## 2. Method & Integrity Constraints

**What "running StratAgent" means.** [Verified] There is no batch harness. An
engagement is an LLM-orchestrated interactive session: the `solve-case` skill
dispatches ~11 subagents (classify → gaps → plan → frame → knowledge → 5
analysts → review → challenge → report → curate). Each run is expensive and
non-deterministic.

**Why 3 genuine cases, not 30 fabricated.** Faithfully executing one full
engagement consumed **13 real subagent dispatches**. Fabricating 30 scored
engagements would have *been* the failure mode this campaign exists to catch
(hallucinated, unsupported output). Three genuine engagements were run and
scored at arm's length; the remaining 27 are **not** claimed or estimated.

**Pilot composition:**

| Case | Archetype | Size / Sector | Pipeline | Gate outcome |
|---|---|---|---|---|
| C04 Northwind Cloud | Market Entry | Enterprise / Tech | **Full** (13 agents) | Reviewer approved · Challenger stands_with_caveats |
| C20 Harbor & Vine | Org Redesign | Mid / Consumer | Lightweight | Challenger stands_with_caveats |
| C09 Halberd Automotive | Cost Reduction | Enterprise / Mfg | Lightweight + **rework** | Challenger needs_rework → **stands_with_caveats** |

Artifacts are on disk: `engagements/northwind-eu-entry/`,
`engagements/harbor-vine-org/`, `engagements/halberd-cost/`.

**Aggregate mean across the 3 pilot cases: ~7.5 / 10.**

### C04 — Northwind (best case, 8.1)
Reframed a one-line prompt into a dual GO+MODE decision plus an EU-vs-APAC
portfolio trade. The `information-gap` agent caught a **real analytical error**
in the classifier (conflating SAM with SOM). The `financial-analyst` didn't
parrot the seeded "EU wins on durability" assumption — it *quantified* that EU
and APAC are a 5-year NPV dead heat and EU only wins at a 6–7yr horizon, and it
surfaced a margin breakeven (21.6%) no upstream agent had. The `challenger`
found the **CLOUD Act sovereignty ceiling** unaided. The report honestly
refused to call EU an NPV winner. Held from 9 by the evidence gap (§5).

### C20 — Harbor & Vine (soft-case probe, 7.1)
Uncovered archetype (no org-redesign cheat sheet). The system **degraded
gracefully** — the classifier flagged the gap honestly, the framework-selector
found org-design frameworks in the vault. Strong structural diagnosis on a
case with *no numbers*. Lowest MECE score (6) because the Challenger correctly
found a **missing option** (a coordination layer over existing P&Ls) the
analyst never considered — and the report incorporated it honestly.

### C09 — Halberd (governance showcase, 7.4)
Two analysts produced an inconsistent cost base (70% vs 55% → $343M gap). With
no Reviewer in the lightweight path, the **Challenger** caught it, caught that
$250M held under *neither* base case, named the load-bearing union clause, and
returned `needs_rework`. The rework reconciled the base, re-cast $250M as a
stretch, and modeled the no-closure floor; the re-challenge verified the fix
**and caught a fresh arithmetic slip**. The single best demonstration that the
governance system does real work.

## 4. Aggregate Statistics

- **Cases run genuinely:** 3 of 30 (10%). **Fabricated:** 0.
- **Total real subagent dispatches:** 32 (C04: 13 full + rework-free; C20: 5;
  C09: 7 incl. 1 rework cycle + re-challenge; plus setup reads).
- **Mean case score:** ~7.5/10. **Range:** 7.1–8.1. **Std dev:** ~0.4 (tight).
- **Governance gate firings:** Reviewer 1/1 approved (with 2 advisories);
  Challenger 3/3 ran; 1 `needs_rework` correctly issued and resolved.

## 5. Common Failure Modes & Findings

**F-1 (Major) — Empty evidence base; the system is an "assumption engine."**
The knowledge vault holds frameworks but **no benchmarks, comps, market sizes,
or precedents** (confirmed by the knowledge-agent: "`data residency`=0 files…
by design the vault holds no numbers," per D-6 reviewer-supplied policy). Every
number in all three engagements is a labeled assumption. This is *honest* (no
hallucination) and matches the stated design, but it means the system cannot
independently ground a claim in external evidence — the "evidence" half of
"evidence over assertion" is entirely user/assumption-supplied. Caps Evidence
Usage at ~6 regardless of case.

**F-2 (Major) — Two divergent framework stores.** The `case-classifier` reads
the plugin's `knowledge/frameworks/*.md` (9 archetype cheat sheets); the
`framework-selector` reads `knowledge-vault/frameworks/*.md` (richer, note-
based). In C20 the classifier declared org-redesign "not covered" while the
vault *did* have org-design frameworks. The system self-corrected downstream,
but two agents holding different views of the framework library is an
architecture defect.

**F-3 (Moderate) — Lightweight pipeline skips the Reviewer, weakening
consistency enforcement.** In C09 the cross-analyst cost-base inconsistency
(RC-3's exact job) reached the Challenger uncaught. The Challenger caught it —
but that is fragile: consistency-checking should not depend solely on the
Challenger. The `challenger` agent's spec also hard-requires a
reviewer-approved precondition, which the lightweight path cannot satisfy; the
agent handled this gracefully ("waived by the lightweight path") but the
contract is internally inconsistent.

**F-4 (Moderate) — RC1 Python reporting layer is off the live path.** The live
report is authored by the LLM `report-writer` as markdown. The deterministic
`render_report` / `check_render_ready` / `validate_consistency` functions (a
significant part of the RC1 build) operate on a populated Python
`EngagementState` that the live pipeline **never populates**. The engagements
pass markdown between agents; no `EngagementState` is instantiated, so the
structural anti-hallucination guards in `packages/reporting` and
`packages/governance` are **not enforced at runtime** — they are exercised only
by the golden-state test fixture. The gates that *did* fire (reviewer/
challenger) are the LLM agents, not the Python `enforce_*` functions.

**F-5 (Minor) — Subagent `Write` tool sandbox-blocked in this environment.**
Every subagent reported it could not write files; the `report-writer` (which
owns report persistence per ADR-005) could not write `report.md`. The
orchestrator persisted all artifacts. The `issue-tree-generator` *did* write
via Bash, so the block is tool-specific. Likely harness-specific, but it means
ADR-005's "report-writer owns the deliverable write" is not literally true here.

**F-6 (Minor) — Latency is high and variable.** Wall-clock is model-bound, not
code-bound. One `risk-analyst` dispatch took ~22 minutes; a full engagement is
tens of minutes of inference. The deterministic layer is microseconds
(`render_report` ~22µs, `validate_mece` trivial) — irrelevant to felt latency.

**F-7 (Minor) — Framework coverage gaps.** Of the 12 benchmark archetypes, the
plugin cheat sheets cover 8; digital transformation, supply chain, customer
experience, due diligence, and product strategy have no dedicated cheat sheet
(some are partially covered in the vault). Uncovered archetypes degrade to the
generic sheet — handled honestly, but with less scaffolding.

## 6. Strengths

- **S-1 Reasoning quality is genuinely senior.** Reframing, sensitivity
  discipline, breakeven-driven recommendations, and non-obvious insight (CLOUD
  Act; the coordination-layer option; the sole-source-plant risk).
- **S-2 Governance works.** The Challenger caught a real inconsistency and an
  over-promise, issued `needs_rework`, and the rework loop resolved it — twice
  catching genuine errors (a $343M base gap and an arithmetic slip).
- **S-3 Zero hallucination; disciplined labeling.** Agents refuse to invent
  data and preserve `[ASSUMPTION]` tags end-to-end into the report appendix.
- **S-4 Honest calibration.** No confidence exceeded 0.7; the Challenger
  actively *lowered* an inflated 0.7 to 0.55–0.60. Reports refuse to overstate
  (EU "is a tie, not a winner"; "$200M base, not a $250M promise").
- **S-5 Graceful degradation.** Uncovered archetype, missing benchmarks, and a
  skipped gate were all handled by disclosing the limitation, not faking past it.
- **S-6 Clean separation of duties.** The Reviewer explicitly asserted it
  authored none of the analysis it gated.

---

## 7. Weaknesses (ranked)

1. Cannot ground claims in external evidence (F-1) — the biggest gap for a
   platform whose tagline is "evidence over assertion."
2. Two framework stores with divergent coverage (F-2).
3. Lightweight pipeline drops the consistency gate; challenger precondition
   contract is inconsistent (F-3).
4. The deterministic reporting/validation layer is not wired to the live
   engagement path (F-4).
5. High, variable latency (F-6).
6. Framework coverage gaps for 4–5 benchmark archetypes (F-7).
7. Subagent write-ownership not enforced in practice (F-5).

---

## 8. Top Improvement Opportunities

1. **Unify the two framework stores** into one source of truth both classifier
   and selector read (fixes F-2).
2. **Wire the Python reporting/governance layer onto the live path** — have the
   orchestrator populate an `EngagementState` from agent outputs and run
   `enforce_render_ready` before accepting a report, so the RC1 anti-
   hallucination guards actually fire at runtime (fixes F-4).
3. **Add a lightweight consistency check to the lightweight pipeline** (a
   scoped Reviewer, or fold RC-3 into the Challenger's contract) (fixes F-3).
4. **Reconcile the `challenger` precondition** with the lightweight path (make
   reviewer-approval conditional on pipeline mode).
5. **Add an optional evidence-retrieval integration** (web/market-data or a
   populated benchmark vault) so at least some numbers can be sourced, not
   assumed (mitigates F-1).
6. **Populate benchmark notes** in the vault for common KPIs/multiples/market
   sizes under the D-6 policy, even as reviewer-approved reference data.
7. **Author framework cheat sheets** for digital transformation, supply chain,
   customer experience, due diligence, product strategy (fixes F-7).
8. **Cap or parallelize slow analyst dispatches**; investigate the ~22-min
   risk-analyst outlier (mitigates F-6).
9. **Persist reports from the owning agent** (or formally reassign persistence
   to the orchestrator in ADR-005) (fixes F-5).
10. **Add cross-analyst numeric reconciliation** as an explicit step when two
    analysts size the same base (the C09 $343M gap).
11. **Surface confidence ceilings in the executive summary**, not just the
    appendix, so readers see the assumption-dependence immediately.
12. **Add determinism affordances** — seed/version the framework set and record
    the assumption ledger so a rerun is structurally comparable.
13–20 (thinner, from single observations): standardize the "no external
    evidence available" disclaimer; add a knowledge-curator run to the pilot to
    test write-back; validate the assumption-ledger breakeven math
    programmatically; add a machine-checkable `[ASSUMPTION]` linter to the
    report; test the FAILED/ABORTED lifecycle paths live; add an engagement-
    level cost/latency budget; test tenant-isolation on a real multi-tenant
    vault; add a golden end-to-end engagement (not just golden state) to CI.

## 9. Performance Summary

| Metric | Observation |
|---|---|
| Full engagement (C04) | 13 subagent dispatches; ~344k subagent tokens; tens of minutes wall-clock |
| Lightweight (C20 / C09) | 5 / 7 dispatches; ~70k / ~103k subagent tokens |

| Memory | Not attributable to the LLM layer; no measurable StratAgent-owned process RSS |

Latency and memory are **not meaningful StratAgent code metrics** — engagement
cost is dominated by model inference. The code that *is* StratAgent's
(validators, gates, renderer) is microsecond-scale and was never the bottleneck.

## 10. Readiness Assessment & Verdict

The core product — structured consulting reasoning with honest evidence
discipline and functioning governance — performed at a genuinely credible
level across all three genuine engagements, including a soft case with no
numbers and a case where the governance loop correctly blocked and then
repaired a flawed analysis. Nothing in the pilot was hallucinated, over-
claimed, or silently wrong; where the system lacked data or a framework, it
said so.

The gaps are real and should be closed before a confident v1.0: the empty
evidence base (F-1), the two divergent framework stores (F-2), the lightweight
pipeline's missing consistency gate (F-3), and — notably — the fact that the
RC1 deterministic reporting/validation layer is not actually enforced on the
live engagement path (F-4). None of these makes the system *wrong*; they make
it *less grounded and less internally coherent* than a 1.0 should be.

### VERDICT: **READY WITH MINOR IMPROVEMENTS**

Scoped honestly: this verdict rests on **3 of 30** genuine engagements. The
reasoning and governance are v1.0-grade; the evidence-grounding and
architecture-coherence issues are "minor" only in that they are improvements,
not correctness failures — but F-1 and F-4 are at the boundary of major and
should not be deferred. **A full 30-case execution is required to convert this
pilot verdict into a production-grade statistical one.**

*This campaign evaluated only. No StratAgent architecture, agents, or features
were modified.*
