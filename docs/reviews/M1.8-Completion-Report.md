# M1.8 Completion Report

# Scope

M1.8 made the Engagement State **durable** on the frozen v1.0 architecture. It
added a new IO-owning sibling package, `packages/persistence`, that saves an
engagement to `engagements/<slug>/` and loads it back into a verified,
append-capable `Engagement` ‚Äî without touching the IO-free `state` package or
any frozen M1.7 seam. Delivered across five approval-gated slices under the
contracts `docs/implementation/M1.8-Design.md` (DD-1..DD-7) and
`docs/implementation/M1.8-Implementation-Plan.md` (PER-001..011). No ADR was
modified; the frozen v1.0 architecture is intact; `packages/state` is
zero-diff across the whole milestone.

# Architectural decision ‚Äî canonical persistence representation

M1.8-S4 surfaced a genuine interaction between two frozen milestones (raised and
approved, not worked around). A live engagement's `committed.state` carries
`projection_version == 0` (built directly by `create()`; `apply()` stamps only
`state_version`), but frozen `verify_pair` (M1.7.4, R14) requires
`projection_version == PROJECTION_VERSION`. **Resolution:** `save()` persists the
canonical projection `project(log)` (provenance normalized to
`PROJECTION_VERSION`), **not** the runtime state. Every persisted
`(log, snapshot)` pair therefore satisfies `verify_pair` on load with no replay
or repair; `verify_pair` remains frozen and unchanged. This is documented as the
canonical persistence representation in `docs/api/Persistence.md` and pinned by
S4-16 and `test_integration.py::test_persisted_snapshot_is_canonical_projection`.

# Out of scope / forward pointers (M1.9)

- **Recovery / repair.** Torn or stale artifacts are *refused* (typed error),
  never repaired. Rebuild-from-log, re-projection on `projection_stale`, and
  torn-write recovery are **M1.9** (DD-5/DD-6).
- **`projection_stale` at load is unreachable by construction** in M1.8 (we
  always persist current-version projections). If `PROJECTION_VERSION` bumps in
  a future milestone, previously-persisted snapshots become stale on load ‚Äî the
  re-projection path for that is M1.9.
- No autosave / implicit / background persistence (DD-7): persistence is always
  explicit via `EngagementStore`.
