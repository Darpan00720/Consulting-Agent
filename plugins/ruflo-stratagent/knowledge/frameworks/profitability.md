# Profitability decline / improvement

## When this applies
"Profit is down," "margins are shrinking," "we need to improve profitability" —
without a stated structural cause yet.

## Core tree
```
Profit = Revenue - Cost

Revenue = Price x Volume (x Mix, if multi-product/segment)
Cost    = Fixed + Variable (or: COGS + SG&A, or per-unit x volume)
```

## Diagnostic sequence
1. **When did it start, and was it sudden or gradual?** Sudden = look for a
   discrete event (lost customer, competitor entry, cost shock, regulation).
   Gradual = look for structural drift (mix shift, creeping cost, share loss).
2. **Decompose the change**: build a bridge from last period's profit to
   this period's, attributing the delta to price, volume, mix, and cost
   buckets. Don't recommend anything before this bridge is built — "costs
   are up" and "volume is down 8% offset by a 3% price cut" call for very
   different fixes.
3. **Segment it.** Is the decline uniform across products/regions/customers,
   or concentrated? A company-wide average can hide one bad segment dragging
   down three healthy ones — segment before averaging.
4. **Check if it's a price, volume, or cost problem** (per the bridge), then
   go one level deeper into the relevant branch:
   - Price: discounting, mix shift to lower-price SKUs, competitive pricing
     pressure, FX.
   - Volume: market shrinking vs. share loss vs. channel/distribution issue.
   - Cost: input cost inflation, fixed cost not flexing with lower volume
     (operating leverage working in reverse), one-time vs. structural.

## Common traps
- Recommending cost cuts before knowing if the problem is actually
  revenue-side.
- Averaging across segments and missing a concentrated problem.
- Treating a one-time cost spike as if it were a structural trend.

## Typical recommendation shapes
Fix pricing/mix, reverse a discrete cause, restructure cost base, exit an
unprofitable segment, or some combination — the diagnostic above should
make the shape obvious before you reach for a generic "cut costs" answer.
